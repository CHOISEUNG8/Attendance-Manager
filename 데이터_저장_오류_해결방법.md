# EXE 파일 데이터 저장 오류 해결 방법

## 발생 가능한 원인

### 1. **쓰기 권한 문제** (가장 흔한 원인)
- EXE 파일이 `Program Files` 같은 보호된 폴더에 있으면 쓰기 권한이 없을 수 있습니다.
- Windows 보안 정책으로 인해 일부 폴더는 쓰기가 제한됩니다.

### 2. **경로 문제**
- EXE 파일 경로에 한글이나 특수문자가 있으면 문제가 될 수 있습니다.
- 경로가 너무 길면 문제가 발생할 수 있습니다.

### 3. **디렉토리 생성 실패**
- 데이터베이스 파일을 저장할 디렉토리가 없거나 생성할 수 없는 경우

### 4. **파일 잠금 문제**
- 다른 프로세스가 데이터베이스 파일을 사용 중인 경우

## 해결 방법

### 방법 1: EXE 파일을 쓰기 가능한 폴더에 배치
- **권장 위치**: `C:\Users\[사용자명]\Documents\근태관리프로그램\`
- 또는 바탕화면에 폴더를 만들어서 배치

### 방법 2: 관리자 권한으로 실행
- EXE 파일을 우클릭 → "관리자 권한으로 실행"
- 단, 이 방법은 매번 관리자 권한이 필요하므로 권장하지 않음

### 방법 3: 코드 개선 (사용자 AppData 폴더 사용)
- 사용자 AppData 폴더에 데이터베이스 파일을 저장하도록 코드 수정
- 이 방법이 가장 안전하고 권장되는 방법입니다.

## 확인 사항

1. **EXE 파일 위치 확인**
   - EXE 파일이 어느 폴더에 있는지 확인
   - 해당 폴더에 쓰기 권한이 있는지 확인

2. **오류 메시지 확인**
   - 정확한 오류 메시지를 확인하면 원인 파악이 쉬워집니다.
   - "Permission denied", "Access denied" 등의 메시지가 있는지 확인

3. **데이터베이스 파일 확인**
   - EXE 파일과 같은 폴더에 `leave_attendance.db` 파일이 생성되는지 확인
   - 파일이 생성되지 않으면 권한 문제일 가능성이 높습니다.

## 권장 해결책

코드를 개선하여 사용자 AppData 폴더에 데이터베이스 파일을 저장하도록 수정하는 것이 가장 안전합니다.
이렇게 하면:
- 쓰기 권한 문제 해결
- 사용자별로 데이터 분리
- Windows 보안 정책 준수
